You are working inside the WFConnect codebase. Implement a full applicant application portal for WFConnect.

GOAL

Create a clean, mobile-friendly applicant form at:

apply.wfconnect.org

This form must allow applicants to submit their information and upload required documents, while making it very easy for staff to review and download files from the CRM/admin side.

--------------------------------------------------
SYSTEM RULES

- Entire system uses Toronto time: America/Toronto
- Do not rely on server local timezone
- Validate all inputs on both frontend and backend
- Store uploads securely
- Make admin access to files easy with one-click download
- Keep structure scalable for future workflow steps

--------------------------------------------------
PART 1 — APPLICANT FORM

Create a public applicant page for apply.wfconnect.org

Required fields:
1. Full Name
2. Address
3. Phone Number
4. What position they are applying for
5. Where they saw the job posting
6. Upload Photo
7. Upload Resume

Form requirements:
- professional and simple UI
- mobile friendly
- clean spacing and large tap targets
- clear required-field indicators
- human-readable validation errors
- disable submit while uploading/submitting
- success state after submission
- prevent duplicate accidental submits

--------------------------------------------------
PART 2 — CANADA-ONLY ADDRESS AUTOFILL

The address field must support autocomplete / autofill for Canada only.

Requirements:
- autocomplete suggestions must only allow Canadian addresses
- user should be able to search naturally
- when an address is selected, save structured values if available:
  - address_full
  - address_street
  - address_city
  - address_province
  - address_postal_code
  - address_country
- block non-Canadian addresses
- address_country must resolve to Canada only
- keep one human-readable full address field as well

If a place API is already available in the stack, use it.
If not, integrate a reliable address autocomplete provider that supports Canada-only filtering.

Frontend behavior:
- show address suggestions as user types
- selecting a suggestion populates the address fields
- if structured parsing fails, still keep address_full
- show a validation error if selected address is outside Canada
