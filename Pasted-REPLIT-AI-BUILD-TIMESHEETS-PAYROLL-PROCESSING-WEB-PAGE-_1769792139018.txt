Goal:

1) View submitted timesheets filtered by a selected payroll period (from the period guide)

IMPORTANT:
- This is for the WEB APP (desktop-friendly), not mobile.

====================================================
3) BACKEND API (EXPRESS) — RBAC ENFORCED
====================================================
Create endpoints:

Periods:
- GET /api/payroll/periods?year=2026
  Returns array of { year, periodNumber, startDate, endDate }

Timesheets:
- GET /api/timesheets?year=2026&period=10&status=SUBMITTED
- GET /api/timesheets/:id (with entries)
- PATCH /api/timesheets/:id/approve   (HR/Admin)
- PATCH /api/timesheets/:id/dispute   (HR/Admin) body { reason }
- PATCH /api/timesheets/:id/update-entry (Admin only, optional)
- NOTE: Do not allow processing if status != APPROVED (unless Admin override)

Payroll processing:
- POST /api/payroll/batches
  body { year, periodNumber }
  Creates or returns existing OPEN batch and auto-includes all APPROVED timesheets for that period.
- GET /api/payroll/batches?year=2026&period=10
- GET /api/payroll/batches/:id (items + totals)
- PATCH /api/payroll/batches/:id/finalize (Admin only)
  Marks batch FINALIZED and marks included timesheets as PROCESSED.
- GET /api/payroll/batches/:id/export.csv (Admin only)
  Exports CSV: Worker Name, Worker Email, Hours, Rate (if uniform or per-entry), Amount, Period, Date Range
