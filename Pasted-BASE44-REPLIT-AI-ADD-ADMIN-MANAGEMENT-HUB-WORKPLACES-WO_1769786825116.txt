4) GPS PROXIMITY FOR TITO MUST BE FUNCTIONAL
====================================================
Requirement:
TITO check-in/out must validate that the worker is within the workplace radius.

Implementation details:
- When worker does Time-In/Time-Out:
  - Fetch device GPS location (with permission flow)
  - Compare distance from workplace lat/lng using Haversine formula
  - If distance <= geofenceRadiusMeters: allow submission
  - Else block submission with clear error:
    “You are not within the required GPS radius of the workplace.”
- Store on the TITO log:
  - recordedLatitude, recordedLongitude
  - distanceMeters
  - gpsVerified boolean
  - gpsFailureReason (optional)
- Always store timestamps in UTC.

Edge cases:
- If workplace has no coordinates yet -> block TITO with message: “Workplace coordinates not configured. Contact admin.”
- If GPS permissions denied -> block TITO with message: “Location permission required for TITO.”
