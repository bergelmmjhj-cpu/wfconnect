====================================================
A) DATABASE (PRISMA) — REQUIRED TABLES / FIELDS
====================================================
1) Workplaces
Create Prisma model `Workplace` with:
- id (cuid)
- name String (required)
- addressLine1 String
- city String
- province String
- postalCode String
- country String @default("Canada")
- latitude Float
- longitude Float
- geofenceRadiusMeters Int @default(150)
- isActive Boolean @default(true)
- createdAt DateTime @default(now())
- updatedAt DateTime @updatedAt

2) Workplace Assignments
Create Prisma model `WorkplaceAssignment` with:
- id (cuid)
- workplaceId String (FK)
- workerUserId String (FK to User)
- status Enum: INVITED | ACTIVE | SUSPENDED | REMOVED  (default ACTIVE)
- invitedByUserId String (FK to User)
- invitedAt DateTime @default(now())
- acceptedAt DateTime? (optional)
- notes String?
- createdAt DateTime @default(now())
- updatedAt DateTime @updatedAt
Add unique constraint: (workplaceId, workerUserId) unique.

3) TITO GPS fields
Extend existing TITO log model (TITOLog or equivalent):
Add:
- workplaceId String? (FK Workplace) OR store via assignment/shift (pick the cleanest)
- gpsLat Float?
- gpsLng Float?
- distanceMeters Float?
- gpsVerified Boolean @default(false)
- gpsFailureReason String?
Also keep UTC timestamps.

Migrations:
- Create and run Prisma migration.
- Update seed script to insert the CAE workplace and sample assignment.

====================================================
C) API ENDPOINTS (EXPRESS) — RBAC ENFORCED
====================================================
All endpoints require JWT auth.

Admin-only:
Workplaces:
- POST /api/workplaces
- GET /api/workplaces
- GET /api/workplaces/:id
- PUT /api/workplaces/:id
- PATCH /api/workplaces/:id/toggle-active

Assignments:
- POST /api/workplaces/:id/invite-worker
  body: { workerUserId, status? }  // default ACTIVE
- GET /api/workplaces/:id/workers
- PATCH /api/workplace-assignments/:id
  body: { status, notes? }
- DELETE /api/workplace-assignments/:id

Worker self:
- GET /api/me/workplaces  // worker can view assigned workplaces

TITO (Worker):
- POST /api/tito/time-in
- POST /api/tito/time-out
Both must:
- require workplaceId OR derive workplaceId from shiftAssignment if your data model supports it
- fetch workplace coords + radius
- read device GPS from request body: { gpsLat, gpsLng }
- compute distance (Haversine)
- allow only if distance <= radius
- store gpsLat, gpsLng, distanceMeters, gpsVerified, gpsFailureReason on the TITO record
Return clear errors if:
- workplace not configured (missing coords)
- gps missing/permission denied
- outside radius
