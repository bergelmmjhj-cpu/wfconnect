Build an ADMIN “Payroll / Timesheet Export” module.

Goal
Admins must be able to download bi-weekly timesheets for payroll, grouped by HOTEL, with payment details ready (eTransfer email + void cheque/banking details). Exports must be separated per hotel and per bi-weekly cutoff period, listing the workers who worked at that hotel during that cutoff.

Core Requirements
1) Access Control
- Only users with role = "admin" can access Payroll / Timesheet Export pages and downloads.
- Block all non-admin access (frontend + backend authorization).

2) UI (Admin)
Create a new Admin page: /admin/payroll
Controls:
- Year selector (default current year; must support 2026)
- Cutoff Period selector (1–26)
- Hotel selector (dropdown + search)
- Optional: “All Hotels” checkbox (generates a zip with one file per hotel)
- Optional filters: worker status (active/inactive), include/exclude cancelled shifts

Results:
- Show a table grouped by hotel:
  - Worker Name
  - Worker ID
  - Dates worked in the cutoff
  - Total hours (or total shifts if hours not tracked)
  - Rate (if available)
  - Gross pay estimate (if rate exists) = hours * rate
  - Payment method: eTransfer
  - eTransfer email
  - Void cheque / banking details (mask in UI; full only in exported file if admin confirms)
  - Notes (optional)

Buttons:
- “Download Timesheet (CSV)”
- “Download Timesheet (XLSX)”
- “Download Payment Summary (CSV/XLSX)” (hotel-level totals + worker payout totals)
- “Download All Hotels (ZIP)” if All Hotels is selected

3) Export Behavior
- Exports are per: (Year, Cutoff Period, Hotel)
- File naming:
  - WorkforceConnect_Timesheet_2026_Period-05_HotelName.csv
  - WorkforceConnect_PaymentSummary_2026_Period-05_HotelName.xlsx
- Payment Summary must include:
  - Hotel name
  - Cutoff date range
  - Worker name/id
  - Total hours/shifts
  - Total pay (if rate exists; otherwise leave blank)
  - eTransfer email
  - Banking/void cheque reference id or details
  - Admin generated date/time

4) Data Requirements / Assumptions
If your current schema differs, implement minimal changes/migrations.

Entities (recommended):
- hotels: id, name, address, isActive
- workers: id, fullName, phone, email, isActive
- assignments/shifts/work_logs: id, workerId, hotelId, shiftDate (or start/end), hoursWorked (nullable), status (completed/cancelled)
- payment_profiles: id, workerId, etransferEmail, voidChequeFileId OR bankInstitution, transit, accountNumber (encrypted at rest), updatedAt

Rules:
- A worker appears in a hotel export if they have at least one COMPLETED work_log within that cutoff date range for that hotel.
- Total hours = sum(hoursWorked). If hoursWorked is null, count shifts and show totalShifts instead.
- If rates exist in your system (worker rate or hotel rate), compute gross pay; otherwise leave pay columns blank but still export payment details.

5) Security / Compliance
- Mask sensitive banking data in UI (e.g., show last 2–4 digits only).
- Exports should include full details only for admin downloads.
- Ensure server-side logging/auditing:
  - who downloaded
  - timestamp
  - cutoff period
  - hotel(s)
  - file type

6) Backend API
Create endpoints (examples):
- GET /api/admin/payroll/cutoffs?year=2026 -> returns list of periods with start/end dates
- GET /api/admin/payroll/hotels -> list hotels
- GET /api/admin/payroll/preview?year=2026&period=5&hotelId=123 -> returns grouped worker rows + totals
- GET /api/admin/payroll/export?year=2026&period=5&hotelId=123&format=csv|xlsx&type=timesheet|paymentSummary -> streams file download
- POST /api/admin/payroll/export/all (optional) -> returns a ZIP for all hotels
