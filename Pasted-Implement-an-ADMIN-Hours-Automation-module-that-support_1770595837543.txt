Implement an ADMIN “Hours Automation” module that supports TWO parallel cycles:
1) WEEKLY invoicing to clients (every Monday)
2) BI-WEEKLY payroll cutoffs (2026 cutoff calendar provided)

Primary purpose: eliminate manual counting of hours by auto-summing approved work logs, and producing invoice/payroll-ready exports grouped per hotel.

Core Principles
- Single source of truth for hours = APPROVED work logs in the app.
- Same underlying aggregation logic, two different “date window” modes:
  A) Weekly window (Mon–Sun) for client invoicing
  B) Bi-weekly cutoff window (period 1–26) for payroll

Data Model (migrate if missing)
1) work_logs (required)
- id
- workerId
- hotelId
- startTime (datetime, tz-aware) OR workDate
- endTime (datetime, tz-aware) OR hoursWorked
- breakMinutes (int, default 0)
- status: submitted|approved|rejected|cancelled
- approvedBy, approvedAt
- createdAt, updatedAt
Hour calculation:
- hoursWorked = max(0, (endTime-startTime) - breakMinutes) converted to hours
- Round to 2 decimals (or nearest 0.25; pick one and apply consistently)

2) payment_profiles (required for payroll exports)
- workerId
- etransferEmail
- voidChequeFileId OR encrypted bank fields
- updatedAt

3) hotels, workers tables as needed.

Timezone / Boundaries
- Use America/Toronto for all boundaries.
- Window is inclusive:
  - start = 00:00:00
  - end = 23:59:59

ADMIN UI: /admin/hours
Create tabs (or segmented control):
TAB 1: Weekly Client Invoicing
TAB 2: Bi-weekly Payroll Cutoffs

TAB 1: Weekly Client Invoicing (Mon–Sun)
Controls:
- Week selector:
  - Default = previous week (since invoices generated Monday for the prior Mon–Sun)
- Hotel selector (single hotel) + “All Hotels”
Preview (grouped by hotel):
- WorkerName, WorkerId
- TotalHours (sum approved logs in the week)
- LogsCount
Exports:
1) Invoice Summary (CSV/XLSX) per hotel:
- Hotel, WeekStart, WeekEnd
- WorkerName, WorkerId, TotalHours, LogsCount
- Optional Amount fields if rates exist
2) Detailed Invoice Backup (CSV/XLSX):
- one row per approved log: date, start, end, breaks, hours

TAB 2: Bi-weekly Payroll Cutoffs (2026 calendar)
Controls:
- Year selector (default 2026/current)
- Period selector (1–26) showing start/end dates
- Hotel selector (single hotel) + “All Hotels”
Preview (grouped by hotel):
- WorkerName, WorkerId
- TotalHours (sum approved logs in cutoff window)
- LogsCount
- Payment info: eTransfer email + void cheque/bank ref (mask in UI)
Exports:
1) Payroll Timesheet (CSV/XLSX) per hotel:
- Hotel, CutoffPeriod, CutoffStart, CutoffEnd
- WorkerName, WorkerId
- TotalHours, LogsCount
- EtransferEmail, VoidChequeOrBankRef
2) Payroll Payment Summary (CSV/XLSX):
- Same as above plus optional pay fields if rates exist

Backend API (admin-only, server-side auth)
Shared aggregation endpoint (recommended):
- GET /api/admin/hours/aggregate
  Params:
  - mode=weekly|cutoff
  - weekStart=YYYY-MM-DD (when mode=weekly)
  - year=2026&period=5 (when mode=cutoff)
  - hotelId=123 OR hotelId=all
Returns:
- perHotel groups:
  - hotel info
  - workers: {workerId, name, totalHours, logsCount, paymentProfile...}
  - hotelTotals: totalHoursAllWorkers, totalLogs

Export endpoints:
- GET /api/admin/hours/export
  Params:
  - mode=weekly|cutoff
  - weekStart=YYYY-MM-DD OR year/period
  - hotelId
  - type=invoiceSummary|invoiceDetailed|payrollTimesheet|payrollPaymentSummary
  - format=csv|xlsx
- POST /api/admin/hours/export/all (optional) -> ZIP across hotels

- weekStart must be a Monday.
- weekEnd = weekStart + 6 days.
- Filter logs where startTime (or workDate) is within [weekStart..weekEnd].
